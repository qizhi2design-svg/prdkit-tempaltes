# PRDKit 原型工具使用指南

## 快速开始

### 1. 创建原型
使用 PRDKit CLI 快速创建新的产品原型：
```bash
prdkit create "原型名称" --template prototype
```

这将创建包含以下文件的完整原型目录：
- `index.html` - 页面结构
- `style.css` - 样式定义
- `script.js` - 交互逻辑
- `marks/` - 标注文档目录

### 2. 添加标注
在 `marks/` 目录下创建 Markdown 文件来标注页面元素：

```markdown
---
id: mark-unique-id
title: 标注标题
element: .css-selector
position: { x: 50%, y: 20% }
---

# 标注内容
使用 Markdown 编写详细说明...
```

### 3. 启动服务
```bash
# 开发模式（支持热重载）
prdkit serve --dev

# 生产模式
prdkit serve
```

## 标注最佳实践

### 选择器类型
- **类选择器**: `.feature-card` - 标注一组相似元素
- **ID选择器**: `#submitBtn` - 标注唯一元素
- **属性选择器**: `[data-mark="title"]` - 自定义标记

### 位置设置
- 使用百分比定位：`{ x: 50%, y: 20% }`
- 相对于视口居中：`{ x: 50%, y: 50% }`
- 靠近元素边缘：`{ x: 10%, y: 10% }`

### 内容组织
1. **概述**：简要说明功能或设计意图
2. **交互说明**：描述用户操作和系统响应
3. **技术实现**：关键代码片段和实现要点
4. **设计考虑**：视觉设计、可访问性等

## 开发技巧

### 热重载
在开发模式下，修改任何文件都会自动刷新页面：
- HTML/CSS/JS 文件变化 → 立即刷新
- Markdown 标注文件变化 → 更新标注内容

### 调试
打开浏览器开发者工具查看：
- Console 日志输出
- 网络请求
- DOM 结构变化

### 样式调试
使用浏览器的元素检查器：
1. 右键点击元素 → 检查
2. 实时修改 CSS 样式
3. 复制满意的样式到 `style.css`

## 常见问题

**Q: 如何关联标注到特定元素？**  
A: 在 frontmatter 中使用 `element` 字段指定 CSS 选择器

**Q: 标注位置如何调整？**  
A: 修改 `position` 字段的 x/y 百分比值

**Q: 支持哪些 Markdown 语法？**  
A: 支持标准 Markdown，包括代码块、列表、表格等

**Q: 如何添加图片？**  
A: 将图片放在原型目录下，使用相对路径引用

## 进阶功能

### 自定义样式
在 `style.css` 中添加自定义样式类：
```css
.custom-highlight {
    background: yellow;
    padding: 2px 5px;
}
```

### 交互增强
在 `script.js` 中添加更多交互逻辑：
```javascript
document.querySelector('.element').addEventListener('click', () => {
    // 自定义交互
});
```

### 响应式设计
使用媒体查询适配不同屏幕：
```css
@media (max-width: 768px) {
    /* 移动端样式 */
}
```
